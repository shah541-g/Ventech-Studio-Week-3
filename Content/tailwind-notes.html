Tailwind Kya Hai, Aur Install Kaise Karein?

Tailwind CSS ek utility-first CSS framework hai — matlab yeh humein pehle se likhi hui CSS classes deta hai (flexbox, padding, colors, borders waghera ke liye) jo hum directly HTML mein apply karte hain, bina khud CSS likhe.

Kaam Kaise Karta Hai

Tailwind CSS core team ne already saari common CSS likh rakhi hai — ek class add karo (jaise flex, p-4), design turant apply ho jata hai. Yeh kisi bhi framework (Next.js, Vue, plain HTML) ke sath kaam karta hai.

content array Tailwind ko batata hai konsi files scan karni hain taake wo unme use hui classes dhoond kar sirf wahi CSS generate kare — is se final CSS file bohot chhoti aur fast rehti hai (JIT — Just-In-Time engine).

Aur Kahan Use Ho Sakta Hai?

  • CLI — Tailwind ka apna installer tool.
  • Framework Guides — Next.js, Vue, Angular, SvelteKit, Gatsby, Phoenix waghera ke liye ready steps.

Traditional approach: pehle HTML likho, phir usay classes se target karke alag se CSS file mein rules likho. Har element ke liye custom CSS likhni parti hai.

Tailwind approach: HTML mein hi directly utility classes add karo — koi separate CSS likhne ki zaroorat nahi.

53 — DARK MODE

Dark Mode — Class Strategy + LocalStorage

// tailwind.config.js
module.exports = {
  darkMode: 'class',   // 'media' bhi option hai (OS preference follow karta hai)
}
<body class="dark:bg-slate-900 dark:text-white">...</body>
/* Jab <html> ya <body> par "dark" class ho, dark: prefix wali sab classes activate ho jaati hain */

Toggle Function (React/Next.js)

const [darkMode, setDarkMode] = useState(undefined);

useEffect(() => {
  if (darkMode === true) {
    document.documentElement.classList.add('dark');
    localStorage.setItem('darkMode', 'true');
  } else if (darkMode === false) {
    document.documentElement.classList.remove('dark');
    localStorage.setItem('darkMode', 'false');
  } else {
    // pehli dafa load — localStorage se pichli preference uthao
    setDarkMode(localStorage.getItem('darkMode') === 'true');
  }
}, [darkMode]);
3-State Trick Kyun

darkMode ko undefined se shuru karna zaroori hai — is se ek teesra "abhi tak decide nahi hua" state milta hai. Agar seedha true/false default rakha jaye, to app hamesha ek theme par load hoga phir localStorage check karke doosri par switch hoga — is se ek "flash" dikhta hai. Undefined state is flash ko avoid karta hai.

Zaroori Order

Toggle button (jo dark class set karta hai) hamesha layout ke sabse top level (<html> ya root <body>) par check hona chahiye, taake children render hone se pehle hi sahi theme decide ho jaye.

54 — ACTIVE RECALL

Practice Questions — Tailwind CSS

Q1. Tailwind ka core philosophy traditional CSS se kaise alag hai?

Traditional mein HTML target karke alag CSS likhte hain; Tailwind mein pehle se likhi utility classes HTML mein directly apply hoti hain, alag CSS likhne ki zaroorat nahi.

Q2. Tailwind config ke content array ka kya kaam hai?

Batata hai konsi files scan karni hain, taake sirf wahi classes final CSS mein generate hon jo actually project mein use ho rahi hain (JIT engine).

Q3. theme.colors mein direct likhna aur theme.extend.colors mein likhna — farq kya hai?

Direct theme.colors Tailwind ke default colors ko overwrite kar deta hai. extend.colors original palette ko rakhte hue naya color upar add karta hai.

Q4. Responsive breakpoint prefix (md:) actually CSS mein kya banta hai?

Ek min-width media query — jaise md: ka matlab hai "768px aur us se upar ke screens par".

Q5. space-x-4 aur simple margin mein practical farq kya hai?

space-x-4 sirf siblings ke beech margin lagata hai, pehle/aakhri element ke bahar nahi — is liye manually calculate karne ki zaroorat nahi parti.

Q6. truncate class actually kin teen properties ka combination hai?

white-space: nowrap, overflow: hidden, aur text-overflow: ellipsis.

Q7. Ring aur border mein bunyadi farq kya hai?

Ring box-shadow-based hota hai, is liye layout space nahi leta (element shift nahi hota) — accessibility-friendly focus indicator ke liye ideal.

Q8. @apply directive kis situation mein useful hai?

Jab same utility classes ka group baar baar repeat ho raha ho — unhe ek custom class (jaise .btn-primary) mein combine karke HTML saaf rakha ja sakta hai.

Q9. Dark mode toggle mein darkMode state ko undefined se shuru karne ki wajah kya hai?

Taake pehli render par galat theme flash na ho — teesra state ("abhi decide nahi hua") localStorage se sahi value load hone tak wait karta hai.

Q10. accent-purple-500 checkbox par appearance-none se better kyun hai?
